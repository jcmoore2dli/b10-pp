Locked Governance Decisions

Student-centric access
Any instructor looks up any student by B10 ID. No class membership. No admin roster maintenance. No CUI risk. Student shares B10 ID voluntarily.
SLT instructor access
Full library — no restrictions. assignmentType = "slt" is a label only. Supplemental pool is additive. Text-prompt tasks are generatively infinite — no supplemental pool needed.
Scaffold system
ESOP + hypothesizing. Two-axis: 6 discourse frames + 7 grammar structures. Four modes: holistic (default) · argument_structure · discourse_frame · grammar_structure · combined.
ILR target values
"2" and "3" only. No "2+" anywhere. 2+ is an OPI rating outcome. mfTier scaffolds within level 3 — all tier 0/1/2 passages are ilrTarget = "3".
Track A level 2 tasks
Assigning narration/description/instructions to Track A = strategic floor prep for 3-level interview gateway. Not remediation.
EXT subtype
EXT is one canonized long-form family — not the umbrella. Future long-form families carry their own corpus types and constraints.
EXT launch
Simultaneous with core passages in Phase 3. Full visibility from day one.
Instructor autonomy
Platform informs sessions — never prescribes them. No gates, no alerts, no pressure.
Recommended path
Orient → Core → Extended · Tier 0 → Tier 1 → Tier 2 as one-tap suggestion. Full library always one tap away.
poolType scope
Audio corpus types only (ORI/COR/EXT). Text-prompt tasks have no poolType — generatively infinite.
MC locked
No MC as primary assessment. ESOP pre-task scaffold only. Track A. Admin-controlled. scaffoldOnly always true.
Audio TTL
Track A only. Deleted after 2 weeks. Transcript + feedback + features persist permanently.
Mobile-first
Full functionality on iPhone and Android. PWA installable. iOS Safari and Android Chrome tested before Phase 3 launch.
Phase 7 placeholder
Plug and play module — 2027–2028. Instructor-defined activities. passageSource = "instructor_created".

Research Direction — Documented, Not Yet a Build Item
Spoken MF Vocabulary Mini Corpus. As attempt transcripts accumulate, a background process could extract mid-frequency vocabulary from attempts scored 3 (Good) or 4 (Excellent) and build a frequency-ranked spoken learner corpus — identifying which MF words actually appear in high-performing ILR 3 spoken production, and which words in the lexicon rarely surface despite being pre-verified.

This would produce something that does not currently exist: a frequency-ranked spoken MF vocabulary list derived specifically from ILR 3-level production in a military English context. Aggregate only — no individual student data. Ethically handled under standard anonymized data use disclosure at platform launch.

Potential value: Informs instruction · surfaces productive vs. dormant MF vocabulary · publishable applied linguistics resource after 12–18 months of data accumulation · improves scaffold and feedback targeting in later phases.

Schema note: No new collections required. Derived from existing /attempts data (transcript + score fields). Would surface as an admin-visible aggregate report.
