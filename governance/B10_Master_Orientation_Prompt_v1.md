## SECTION 2 — B10 SYSTEM ARCHITECTURE (THREE LAYERS)

B10 consists of THREE architecturally distinct and non-collapsible layers:

### LAYER 1: ORIENT
- Purpose: Introduce three core ILR-3 discourse organization patterns using lexically simple (CEFR B0–B1) language
- Three patterns ONLY:
  - Pattern 1: Trend → Complication → Reassessment
  - Pattern 2: Phenomenon → Explanation → Implication
  - Pattern 3: Example → Reinterpretation → Broader Meaning
- One pattern per passage, no blending
- CEFR B0–B1 lexical ceiling (non-negotiable)
- Full structural integrity maintained
- Does NOT use: Tier labels, MF counts, CI-hooks, PIL, EXT logic
- Exists to stabilize discourse recognition PRIOR to lexical calibration

### LAYER 2: CORE (70–87 words)
- Purpose: Lexical Calibration Layer
- Word count: 70–87 words ONLY
- Structure: 4–5 sentences, single paragraph, one algorithm
- Counter-intuitive element REQUIRED
- ≥2 complex sentences, ≥3 grammar features, 2–4 discourse markers
- Anchoring Stabilization v1.1 enforced (see Section 4)
- 30-check SPEC validation required

**CORE Tier Doctrine (LOCKED):**
Tier is defined ONLY by MF (mid-frequency) word count. Nothing else. Structure is IDENTICAL across all tiers.

- **Tier 0:** MF count 1–2 maximum. Pattern exposure with minimal lexical strain. Anglo-Saxon vocabulary preferred. Structurally complete.
- **Tier 1:** MF count 3–6. ≥1 MF in Sentence 1. ≥1 MF in final sentence.
- **Tier 2:** MF count 6–10. Full lexical density band.

CORE does NOT use: PIL, EXT bands, CEFR ceilings.

### LAYER 3: EXT (120–185 words)
- Purpose: Processing Density Layer (NOT a tier extension — a separate architectural dimension)
- Introduces: Length Bands + Processing Intensity Level (PIL) + Anti-Stacking Lexical Control

**EXT Length Bands:**
- EXT-S: 120–140 words
- EXT-M: 141–165 words
- EXT-L: 166–185 words

**PIL (Processing Intensity Level):**
- PIL-F: Low density
- PIL-M: Mid density
- PIL-H: High density
PIL controls dependency chaining, inference load, ellipsis density, reversal timing, recovery difficulty.

**Anti-Stacking Rule (LOCKED):**
Difficulty may increase through length (EXT) OR processing intensity (PIL) OR lexical density (LEX-A/LEX-B) — but NOT all simultaneously at maximum. LEX-B is EXCLUDED from PIL-H. This preserves ILR-3 ceiling integrity.

**EXT 72-Item Mapping (v2.0):**
- All EXT-S buckets (1–3): LEX-A only regardless of PIL
- EXT-M / PIL-F and PIL-M (buckets 4–5): LEX-A and LEX-B permitted
- EXT-M / PIL-H (bucket 6): LEX-A only
- EXT-L / PIL-F and PIL-M (buckets 7–8): LEX-A and LEX-B
- EXT-L / PIL-H (bucket 9): LEX-A only

EXT does NOT use: Tier 0/1/2 labels, CORE MF placement rules, Anchoring Stabilization v1.1.

## SECTION 4 — ANCHORING STABILIZATION v1.1 (CORE ONLY)

By end of Sentence 2, a CORE passage MUST include:
1. At least one **Explicit Referential Anchor** (named professional group, institutional actor, specific population, concrete setting, measurable phenomenon, observable event)
2. At least one **Concrete Domain Anchor** (specific location/context, identifiable group, physical system, policy mechanism, measurable variable, observable practice)

Actor-first construction is OPTIONAL. Entry diversity is REQUIRED.

**Prohibited entry patterns:**
- Pure abstraction without referent by S2
- Concept stacking without domain grounding
- "It is believed…" constructions
- Generic macro-process openings without context

**Entry diversity types required (vary across batches):**
1. Actor-first
2. Setting-first
3. Event-first
4. Measurable-variable-first
5. System-first (with concrete referent)
6. Population-first

## SECTION 8 — SCORING RUBRIC (CORE PARAPHRASE TASK)

The paraphrase scoring rubric (4-point scale) used in B10-PP (B10 Practice Platform):

**Score 4 — Excellent:**
Clearly conveys the passage's central claim; accurately expresses key relationships (cause/effect, unexpected findings, contrasting expectations); includes all essential explanatory logic; presents ideas coherently without distortion.

**Score 3 — Good:**
Conveys the general main idea OR one central finding accurately; includes at least one major causal or explanatory relationship correctly; shows overall accurate understanding even if some essential relationships are missing; is coherent and does not distort meaning; demonstrates substantial but incomplete coverage of the passage's logic.

**Score 2 — Partial:**
Mentions only the topic or an isolated detail; does not clearly convey the passage's main idea or central findings; shows only partial comprehension; omits or confuses key relationships; lacks a coherent explanation of the passage's logic.

**Score 1 — Inaccurate:**
Misrepresents the meaning; introduces major errors; shows minimal comprehension.

**Scoring agent:** Claude API (Anthropic). The system prompt for the scoring call will embed: (a) the full passage text, (b) this 4-point rubric, (c) ILR-adjacent criteria for the specific tier. Claude returns structured JSON: score (1–4), score label, strengths, gaps, specific language feedback.

## SECTION 9 — TWO TOOLS UNDER DEVELOPMENT

### TOOL A: B10-PP — B10 Practice Platform (Snorkl Replacement)
**Goal:** A Progressive Web App hosted on GitHub Pages that replicates and extends Snorkl functionality, tuned specifically to the B10 corpus and rubric.

**Student task types (NO multiple choice):**
- Oral paraphrase of passage (primary task — MVP Phase 1)
- Extended Supported Opinion (ESO) spoken response to policy prompt (Phase 2)
- Hypothetical/conditional spoken response (Phase 2)

**Stack:**
- Frontend: React + Vite PWA on GitHub Pages
- Audio: Pre-generated Azure TTS MP3s served statically (one standard neural voice)
- Student recording: Web Audio API (browser microphone)
- Transcription: OpenAI Whisper API ($0.006/min)
- Scoring: Anthropic Claude API with B10 4-point rubric system prompt
- Storage: Firebase (Firestore) — permanent cloud datastore; localStorage = temporary buffer only
- Cost estimate: ~$0.01 per student session

**Scoring philosophy:**
- No multiple choice
- Claude is the scoring agent
- Rubric is the B10 4-point paraphrase scale (see Section 8)
- Scores are raw 1–4 only in MVP — no ILR level estimate, no B10 performance label

**Current status:** B10-PP Spec v0.1 fully drafted (13 sections). GitHub repository not yet created. Build not yet started. Pre-build environment setup is the immediate next task.

### TOOL B: B10 Prosody Pitch Contour Visualizer
**Goal:** A browser-based tool that displays student pitch contour (F0) overlaid on a native speaker reference, enabling visual prosody feedback for segmental and suprasegmental practice.

**Target features (phased):**
- Phase 1 (MVP): Pitch contour display — student F0 curve vs. native speaker reference curve, overlaid on canvas
- Phase 2: Segmental phrase context practice (linking, blending)
- Phase 3: Focus and primary phrase stress feedback (intensity + duration peak on correct syllable)

**Stack:**
- Pitch extraction: Python `parselmouth` (Praat wrapper) for reference recordings; Web Audio API AnalyserNode for real-time student input
- Visualization: HTML5 Canvas overlay
- Reference recordings: Native speaker MP3s from B10 passage corpus
- Integration: Potential tab within B10 Listening Lab PWA (same GitHub Pages deployment)

**Current status:** Architecture defined. Build not yet started. Pitch contour module is the first build target.
