Four Focus Modes
holistic
Default. Claude evaluates all ESOP dimensions equally. No scaffold shown to student. PSU always implicitly evaluated.
argument_structure
Claude foregrounds PSU evaluation: point → support → analysis arc. Plain cue shown to student. No additional selection needed.
discourse_frame
Claude gives primary attention to selected frame. Requires primaryFrame. Holistic evaluation continues alongside.
grammar_structure
Claude gives primary attention to selected structure. Requires primaryStructure. Holistic evaluation continues alongside.
combined
Claude attends to both selected frame and structure. Requires both. Student sees both plain cues.
SLT Instructor Access — No Restrictions
SLT instructors have full library access. assignmentType = "slt" is a label only — it does not restrict what they can assign. The supplemental pool (poolType = "supplemental") is additive — more options, not fewer. Students receiving only SLT need unrestricted Track A access. Text-prompt tasks (ESOP, hypothesizing, narration, description, instructions) are generatively infinite — no supplemental pool needed for these.

ILR Target — Canonical Definition

Critical — ilrTarget has exactly two values: "2" and "3". No "2+" anywhere.
2+ is an OPI rating outcome — not a task category. No passage trains "2+ behavior." Passages train level 2 or level 3 tasks. The 2+ rating emerges from a 3-level interview when the student shows a solid level 2 floor but a ceiling at level 3.

mfTier scaffolds within level 3 — all tier 0/1/2 passages are ilrTarget = "3". Track A level 2 task assignments are strategic floor prep for the 3-level interview gateway — not remediation. Most Track A students will never receive them.

EXT — Subtype, Not Umbrella

Architectural Position — Locked
EXT = one canonized long-form listening family. EXT ≠ the universal template for all extended listening content. EXT enforces constraints appropriate to its purpose — single controlling idea, counter-intuitive hinge, no lists, hard-gated length bands, strict recovery logic. These constraints are correct for EXT. They do not generalize to all future long-form content.

Future long-form families may include short lectures, briefings, multi-point discourse, or content with lists and enumeration. These will carry their own corpusType values and metadata fields. Adding a future long-form family requires no refactoring of existing collections — zero impact on EXT documents.

Access Control

Firestore Security Rules
Student lookup: Any authenticated instructor reads any student by B10 ID — all attempts, notes, assignments, scores, feedback. No class membership required.

Passage visibility (students): Student reads passage only if passage.track == student.track OR passage.track == "both". Enforced at database level, not UI level.

track immutability: student.track write-protected after creation. Admin override requires non-empty trackOverrideNote.

Groups: Private to owning instructor. Other instructors cannot read another's groups.

Notes: Any instructor who looks up a student may write and read notes. Visible to all instructors who look up that student and to admin.

Design Notes

Student-centric model
Student is the primary unit. B10 ID is the access key. Any instructor looks up any student they work with. No class membership, no admin roster, no CUI risk.
SLT access
Full library — no restrictions. assignmentType = "slt" is a label only. Supplemental pool is additive. Text-prompt tasks are generatively infinite — no supplemental pool needed for these.
Scaffold system
Two-axis: 6 discourse frames + 7 grammar structures. Default is holistic. Claude receives technical spec. Student receives plain-language cue only. PSU always implicitly evaluated — argument_structure foregrounds it.
claudeFeedback scope
Written to every attempt from Phase 3 launch. Addresses grammar, structure, MF vocab, abstraction, discourse, fluency. Scaffold-aware when scaffoldConfig present. Feedback quality is not deferred. Analytics are.
MF — passage input
Pre-verified externally in Notion. Claude does not classify passage vocabulary. mfTier = human-verified label on ORI/CC. EXT: mfTier = "n/a" per EXT-SPEC-DELTA v1.3.
MF — student output
Phase 4 feature. Claude evaluates student transcript against pre-verified lexicon. Populates mfVocabUsed / mfVocabMissed. Claude as rater of student production — not corpus classifier.
ILR target values
"2" and "3" only. No "2+". 2+ is an OPI rating outcome. mfTier scaffolds within level 3 — all tier 0/1/2 passages are ilrTarget = "3".
EXT subtype
EXT is one canonized long-form family — not the umbrella. Future long-form families carry their own corpus types. Zero refactoring needed to add them.
Future task types
reported_speech_2 (ILR 2 · concrete topic) and reported_speech_3 (ILR 3 · abstract topic) are future taskType values. Not yet in enum — added when corpus is ready. No migration needed.
Instructor autonomy
Platform informs sessions — never prescribes them. No gates, no alerts, no pressure. If an instructor feels boxed in, they will not use it.
Audio TTL
Track A only. Deleted after 2 weeks. Transcript + feedback + features persist permanently.
Mobile-first
Full functionality on iPhone and Android. PWA installable. iOS Safari and Android Chrome tested before Phase 3 launch.
Session note summary
On-demand only — instructor clicks "Summarize notes" to generate. Claude synthesizes all session notes for that student into 3–5 sentences. Result cached in students.studentSummary until instructor clicks Refresh. Timestamp shown so instructor knows how current it is. Never auto-generates on note write — that would accrue cost without corresponding benefit. The summary serves orientation at session start, not logging at session end.
Privacy / CUI
B10 ID is system-generated — not linked to school ID. Student shares voluntarily. No roster. No sensitive institutional data in the system.
