## FILES YOU WILL CREATE

1. services/sanitizeTranscript.js
   - Stub sanitizer
   - Removes placeholder patterns ([NAME], [LOCATION] etc.)
   - Logs what was redacted (category only, not content)
   - Returns sanitized text and sanitization status
   - Fail-closed: if sanitization errors, returns failure status

2. services/pipeline.js
   - Accepts raw transcript, task type, passage, arcAnchor
   - Calls sanitizeTranscript.js
   - If sanitization fails, returns error — does NOT call scorer
   - If sanitization passes, calls scoreResponse.js
   - Returns structured result object

3. testPipeline.js
   - Tests full flow end to end
   - Test A: clean transcript (no redaction needed)
   - Test B: transcript with mock PII markers
   - Test C: simulated sanitization failure

## STEP BY STEP

### Step 1 — Create sanitizeTranscript.js

Location: services/sanitizeTranscript.js

Purpose:
- Accept raw transcript text
- Run pattern matching for known identifier types
- Replace matches with redaction markers
- Return:
  {
    sanitized: true/false,
    text: <sanitized transcript text>,
    redactions: <count of items removed>,
    error: <null or error message>
  }

Patterns to catch in stub (rule-based, not NER):
- Proper name patterns (Captain, Sergeant, Dr., Mr., Mrs.)
- Country/city references (expand later with NER)
- Email patterns
- Unit designators (e.g. 3rd Battalion, 101st)

Fail-closed behavior:
- If the function throws, return sanitized: false
- Pipeline must check sanitized status before calling scorer

---

### Step 2 — Create pipeline.js

Location: services/pipeline.js

Purpose:
- Single entry point for the full evaluation flow
- Connects sanitization and scoring
- Returns a normalized result object

Input:
{
  taskType: 'PARA' or 'EXT',
  passageText: string,
  rawTranscript: string,
  arcAnchor: object or null
}

Output (success):
{
  status: 'scored',
  sanitizationPassed: true,
  redactions: <count>,
  score: <integer>,
  score_label: <string>,
  strengths: <string>,
  gaps: <string>,
  language_feedback: <string>,
  transcript_note: <string>
}

Output (sanitization failure):
{
  status: 'sanitization_failed',
  sanitizationPassed: false,
  error: <string>
}

Output (scoring failure):
{
  status: 'scoring_failed',
  sanitizationPassed: true,
  error: <string>
}
